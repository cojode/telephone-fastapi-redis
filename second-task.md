# Задание второе (теоретическое)
> Ты наверняка уже видел нашего HR-бота в Telegram. И, вероятнее всего, понял, что в нём чего-то не хватает.

> Потрать 20–30 минут и сформулируй, чего именно, на твой взгляд, ему недостаёт. Представь, что следующий бот, который будет встречать кандидатов, будет разработан именно тобой.

## 1. Недостаток предварительного предупреждения о начале отсчёта времени

После нажатия на кнопку «Получить тестовое задание» бот сразу сообщает о запуске таймера. Для кандидатов это может быть неожиданностью: не все готовы начать выполнение прямо в тот момент.

Было бы лучше заранее вывести короткое уведомление, например:

`«После нажатия кнопки начнётся отсчёт времени. Убедитесь, что вы готовы приступить.»`

Это снимет лишний стресс и сделает процесс более прозрачным.

## 2. Неочевидный формат ожидаемого ответа

Сейчас бот сообщает:

>«...Пожалуйста, по готовности пришлите ответ ссылкой на Google Docs одним сообщением с результатами выполненного тестового задания...»

Но в самом тестовом задании указано:

> * Первое задание — разместить в Git
> * Второе — можно в Git в формате MD или в Google Docs

В итоге возникают разночтения:

- Нужно ли обязательно использовать Google Docs для второго задания?

- Можно ли прислать две разные ссылки (Git + Google Docs), если бот просит «одним сообщением»?

- Как бот будет реагировать на альтернативный формат (например, MD в Git), который в условиях допускается?

Для кандидата это добавляет ненужной неопределённости, а для бота — риск некорректной обработки результатов.

## 3. Недостаток пояснений о структуре итогового сообщения

За счёт формулировки «одним сообщением» не совсем ясно:

- Достаточно ли отправить одно сообщение, но с двумя ссылками?

- Или подразумевается одна ссылка, которая должна содержать всё?

- Можно ли отправлять комментарий к решению или только ссылки?

Уточнение этих деталей снизило бы количество ошибок и исключило ситуации, когда кандидат формально выполняет задание, но нарушает скрытые ожидания бота.
